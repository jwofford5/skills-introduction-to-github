Working Memory & Cognitive Architecture

From Cognitive Load Theory — Sweller, Ayres & Kalyuga (2011)

Why Cognitive Architecture Matters

Without knowledge of how the human mind processes information, instructional design is blind. We can observe that some teaching methods work and others do not, but without an underlying model of cognition we cannot explain why, predict when they will generalise, or systematically improve them.

Cognitive Load Theory is built on a specific model of human cognitive architecture — the structures and processes that govern how we take in, hold, and permanently store knowledge. That architecture has two central components: long-term memory and working memory.

Long-Term Memory: The Unlimited Store

Long-term memory is the foundation of expertise. It holds an essentially unlimited amount of information for an unlimited duration. Everything you know — from how to ride a bicycle, to the rules of grammar, to the layout of your home — lives there.

Information in long-term memory is organised into schemas: mental structures that group related elements into a single unit. A chess grandmaster does not see 32 individual pieces on a board — they see familiar formations, each stored as one schema. This bundling is what enables expert performance.

The ultimate goal of instruction is to help learners build rich, accurate schemas in long-term memory. Once a schema is stored, it can be retrieved into working memory and used almost effortlessly.

Working Memory: The Bottleneck

Working memory is where conscious thought happens. It processes incoming information from the senses and retrieves information from long-term memory. But when handling novel information — material that has not yet been stored as a schema — working memory is severely constrained in two ways:

~4
items (Cowan, 2001)
Storage capacity for novel items. Earlier estimates were 7±2 (Miller, 1956), but lower figures emerge for active processing.
2–3
novel elements
Processing capacity. When elements must be organised or compared, the usable limit is even lower than storage capacity.
<20s
without rehearsal
Duration. Novel information disappears in under 20 seconds unless actively rehearsed (Peterson & Peterson, 1959).

These limits are not design flaws. They arise from a fundamental constraint: when encountering something genuinely new, the mind must explore possible ways to organise it. With just three elements there are only 6 permutations to consider; with ten elements there are over 3.6 million. Keeping working memory small keeps this combinatorial explosion manageable.

Working memory can also be equated with consciousness itself: you are only directly aware of what is currently in working memory. Everything else in long-term memory exists below the threshold of awareness until retrieved.

Two Channels, Not One

Working memory is not a single bucket. Following Baddeley's influential model, it contains at least two partially independent processors that correspond to the modality of incoming information:

👁

Visual-Spatial Sketchpad

Handles images, diagrams, spatial layouts, and written text processed visually.

🔊

Auditory / Phonological Loop

Handles spoken words, sounds, and inner speech (mentally "sounding out" text).

Because the two channels are partially independent, presenting information across both simultaneously can expand effective working memory capacity. A diagram explained by spoken narration uses both channels; a diagram with identical on-screen text competes with itself in the visual channel and can overload it — a key insight behind several cognitive load effects.

The Expert Advantage: When Limits Disappear

The capacity and duration limits of working memory apply only to novel information obtained from the environment. When working memory retrieves previously organised schemas from long-term memory, those limits effectively vanish.

A schema, however complex, counts as a single element in working memory. An expert radiologist scanning an X-ray is not processing hundreds of individual features; they retrieve a handful of high-level patterns stored as schemas. This is why expertise makes previously impossible tasks feel trivial — not because working memory has grown, but because long-term memory now does most of the work.

This has a direct instructional implication: the more schemas a learner builds, the freer their working memory becomes to handle genuinely new material. Instruction that accelerates schema formation is instruction that expands functional cognition.

Practical Examples

📞

Dialling an Unfamiliar Phone Number

You look up a number you've never seen, then walk to the phone. By the time you get there, it's gone. This is working memory's duration limit (~20 s) in action. You solve it by rehearsing the number aloud — keeping it alive in the phonological loop. With a number you dial every day, no rehearsal is needed: a schema in long-term memory handles it instantly.

🚗

Learning to Drive vs. Driving as an Expert

A new driver must consciously track steering, mirrors, pedals, speed, and traffic simultaneously — often hitting the processing limit of 2–3 novel elements. An experienced driver has bundled all those sub-tasks into automatic schemas, freeing working memory to hold a conversation, navigate, or react to the unexpected. Nothing physiological changed; long-term memory did the work.

♞

Novice vs. Expert Chess Player

In a classic experiment, chess masters and beginners were briefly shown a mid-game board, then asked to reconstruct it. Masters recalled nearly every piece; beginners remembered only a handful. When the same pieces were placed randomly (no real game pattern), masters were no better than beginners. Masters don't have better working memories — they have thousands of board-position schemas in long-term memory, each retrieved as a single chunk.

🗺

Reading a Map While Listening to Directions

Studying a road map (visual channel) while a passenger reads directions aloud (auditory channel) is manageable. Trying to read the directions on-screen and study the map simultaneously forces both tasks into the same visual channel, rapidly overloading it. Using both channels in parallel exploits the partial independence of working memory's processors and reduces overall cognitive load.

Instructional Takeaways

  • Respect the bottleneck. Novice learners cannot process more than 2–3 novel concepts simultaneously. Break complex material into smaller steps and present them sequentially rather than all at once.
  • Build schemas deliberately. The goal of every lesson is to move organised knowledge into long-term memory. Activities that build schemas — worked examples, guided practice, spaced retrieval — directly expand the learner's effective cognitive capacity over time.
  • Use both channels. Where possible, pair visual material (diagrams, animations) with spoken explanation rather than on-screen text. This distributes the load across both processors and avoids overloading the visual channel.
